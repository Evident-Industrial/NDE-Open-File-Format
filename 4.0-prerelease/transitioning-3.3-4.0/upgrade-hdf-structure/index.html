    HDF structure upgrade guide

    Terminology¶

    Note the correlation between terms used in version 3.3 and in new terms used in version 4.0.

    Version 3.3 Version 4.0
    DataGroupId GroupId
    DatasetId DatasetId
    DatasetType DataClass

    Convert DatasetTypes to DataClasses¶

    The following correlations need to be made between previous DatasetTypes and new DataClasses:

    DatasetType DataClass
    Amplitude AScanAmplitude
    Status AScanStatus
    FiringSource FiringSource
    Value (for Amplitude TFM) TFMAmplitude
    Value (for PCI) TFMPhase

    Note that for TFM datasets, you will need to determine through the JSON Setup dataset whether it was an Amplitude TFM or a PCI so as to specify the correct DataClass.

    Update datasets paths¶

    It results in the following correlation table for each dataset path. Note that TFM datasets were previously stored under a dedicated sub-path which is now removed for standardization.

    Type Previous datasets path New datasets path
    PA /DataGroups + /[DataGroupId] + /Datasets + /[DatasetId] + /[DatasetType] /Groups + /[GroupId] + /Datasets + /[DatasetId-DataClass]
    TFM /DataGroups + /[DataGroupId] + /Datasets + /[DatasetId] + /TFM + /[DatasetType] /Groups + /[GroupId] + /Datasets + /[DatasetId-DataClass]

    The following examples are provided for each DatasetType/DataClass for better understanding:

    Previous dataset path New dataset path
    /DataGroups/0/Datasets/0/Amplitude /Groups/0/Datasets/0-AScanAmplitude
    /DataGroups/0/Datasets/0/Status /Groups/0/Datasets/1-AScanStatus
    /DataGroups/0/Datasets/1/FiringSource /Groups/0/Datasets/2-FiringSource
    /DataGroups/1/Datasets/0/TFM/Value /Groups/1/Datasets/0-TFMAmplitude
    /DataGroups/1/Datasets/0/TFM/Status /Groups/1/Datasets/1-TfmStatus
    /DataGroups/1/Datasets/0/TFM/FiringSource /Groups/1/Datasets/2-FiringSource
